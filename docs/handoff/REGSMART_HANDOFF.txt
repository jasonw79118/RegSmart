REGSMART HANDOFF — PHASE 1 UI EXPANSION / ENTERPRISE MODE

Current build direction
- RegSmart is being built as a Salesforce-style, cloud-based compliance operating system.
- Starting mode is FULL STRUCTURE, not simplified mode.
- Architecture assumes lead bank + subsidiaries + business units from the beginning.
- Deliverables should be provided as downloadable ZIP files, not code pasted in chat.
- Include a handoff document automatically when phases get larger or when moving to a new chat.

What this pack adds
- Expanded web shell with enterprise / multi-entity positioning.
- Left navigation organized into workspaces and structure sections.
- Top bar indicating enterprise mode.
- Dashboard expanded to show entity exposure, issue source mix, high-priority issues, and active reviews.
- Entities page added for legal/entity structure.
- Issues page expanded with entity, source, severity, status, exposure, and due date columns.
- Issue detail page expanded with summary metrics, action plan, evidence section, and timeline.
- Audits & Reviews page expanded with linked issue counts and review detail routing.
- Audit detail page added.
- User administration page expanded to reflect role model and provisioned users.
- Shared mock data layer added to support future API integration.
- Shared section-card and status-badge components added for reusable UI growth.

Architecture direction
- Frontend stays modular and component-based.
- Backend remains separate for future API growth and secure bank deployment patterns.
- APIs should remain versioned under /api/v1.
- Entity-aware design should remain in place even if a future customer uses only one bank entity.
- Role/scope model should evolve toward institution, entity, department, and workflow access.
