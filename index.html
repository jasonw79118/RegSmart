Help / User Guide

  • What is an issue? A documented compliance, audit, operational, or control weakness that requires ownership, response, evidence, validation, and closure.
  • Why keep issues open? An issue should stay open until corrective action is implemented and independently validated.
  • Examiner-ready expectation: Each issue should show source, statement, severity, owner, due date, notes, and a clear chronology of movement through the lifecycle.
  • Example: A UDAAP disclosure timing exception may start in Compliance Review, move to Remediation after ownership assignment, then to Validation when updated disclosures and training evidence are provided.
  • Drawer workflow: Click an issue row to review the full record, update its state, add notes, and save.
  • Version control: The active build version remains visible in the bottom-left and should match the ZIP package version exactly.
