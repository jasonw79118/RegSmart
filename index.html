Help / User Guide

  • What is an issue? An issue is a documented control weakness, process gap, policy exception, or deficiency identified by audit, compliance, management, examiners, or third parties that requires ownership, action, and evidence.
  • How the lifecycle works: Identified → Logged → Assigned → Remediation → Evidence Submitted → Validation → Closed. Use the slide-out issue panel to move between steps.
  • Examiner-ready expectation: each issue should show a clear statement of the problem, responsible owner, corrective action, target date, and evidence supporting closure.
  • Example compliance issue: A fair lending timing exception found in file review should remain open until root cause is documented, retraining is complete, and validation confirms the process now operates correctly.
  • Example BSA issue: A CTR processing gap should remain open until the sample set, retraining records, and corrected monitoring evidence are attached and reviewed.
  • Version control: the active build version remains visible in the bottom-left so GitHub Pages deployment can be confirmed visually after each push.
